## ‚ö° FASTEST PATH TO DEPLOYMENT (10 Minutes)

### Step 1: Create GitHub Account (2 minutes)
1. Go to [github.com](https://github.com)
2. Click "Sign up"
3. Use email: `lizzycreatescoolstuff@gmail.com`
4. Create password and username
5. Verify your email

### Step 2: Create Repository (1 minute)
1. Click **+** (top right) ‚Üí **New repository**
2. Name: `pixels-dojo-wiki`
3. Choose **Public**
4. Check "Add README"
5. Click **Create repository**

### Step 3: Upload Files (3 minutes)
1. In your repository, click **Add file** ‚Üí **Upload files**
2. Drag ALL files from this folder into GitHub
   - Make sure you include the folders: `css/`, `js/`, `pages/`, `images/`
3. Commit message: "Initial wiki upload"
4. Click **Commit changes**

### Step 4: Enable GitHub Pages (2 minutes)
1. Click **Settings** (top menu)
2. Click **Pages** (left sidebar)
3. Under "Source", select **main** branch
4. Click **Save**
5. Wait 2 minutes for site to build

### Step 5: Add Your Images (2 minutes)
1. Go to `images/` folder in your repository
2. Upload your logo files:
   - `pixels-dojo-logo-round.png`
   - `banner.png`
3. Commit changes

‚úÖ **DONE!** Your site will be live at: `https://YOUR-USERNAME.github.io/pixels-dojo-wiki/`

### Common Issues:

**Site not loading?**
- Wait 2-5 minutes after enabling Pages
- Make sure repository is Public
- Check Settings ‚Üí Pages for errors

**Images not showing?**
- Make sure images are uploaded to `images/` folder
- Check file paths in HTML: `../images/filename.png`
- Filenames are case-sensitive!

**Changes not appearing?**
- Wait 2 minutes for GitHub to rebuild
- Clear browser cache (Ctrl+Shift+R)
